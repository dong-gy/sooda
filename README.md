### 🚀 Sooda - 动态 API 编排引擎

**告别繁琐的编译与部署，像搭积木一样构建你的 API。**

**核心价值：** 将 API 开发时间从“小时级”降至“分钟级”，实现真正的**零停机热更新**。

---

### 🤔 背景与痛点

你是否也有过这样的经历？

产品经理：“我们需要加一个简单的接口，查一下用户数据就行。”
你（内心崩溃）：“好的，等我打开 IDE -> 写路由 -> 写逻辑 -> 打包 -> 走 CI/CD -> 部署……大概半小时后上线。”

传统的 API 开发流程过于“沉重”。哪怕只是几行代码的改动，也必须走完整套编译、打包、部署流程。这不仅浪费时间，还打断了开发者的思维连贯性。

### 🧊 什么是 Sooda？

**Sooda** 的名字源自“瞬间解决”的畅快感。我们旨在为后端开发提供一种像前端路由一样灵活的**动态 API 构建方式**。

Sooda 是一个基于**拖拽画布**的低代码 API 编排引擎。你不需要编写传统的 Java/Node.js 服务并重启应用，而是通过可视化界面直接定义 API 的执行逻辑。

**核心工作流：**

1. **拖拽**：在画布上拖拽“数据库查询”、“逻辑判断”、“HTTP 调用”等节点。
2. **连线**：定义数据的输入与输出流向。
3. **发布**：点击保存，Sooda 引擎实时解析逻辑，API **立即生效**，无需重启服务。

### ✨ 核心特性

- **⚡ 实时生效 (Real-time)**
改完即上线。保存流程图的那一刻，对应的 API Endpoint 就可以被调用，彻底告别 `mvn package` 和服务器重启。
- **🧩 可视化编排 (Visual)**
基于 React + AntV G6 打造的丝滑画布体验。把复杂的代码逻辑转化为直观的节点图，让“逻辑”看得见。
- **🔌 丰富的连接器 (Connectors)**
内置通用数据库连接器（MySQL, PostgreSQL, MongoDB）和 HTTP 客户端。未来将支持 Redis、Kafka 等更多中间件。
- **🛡️ 生产级就绪 (Production-Ready)**
支持 JWT/OAuth 鉴权、请求限流、全链路监控以及版本回滚。不仅开发快，运维也安心。
- **🧱 混合开发模式**
不是全盘替代。你可以在 Sooda 中处理简单的 CRUD，同时保留核心业务在传统代码中，两者完美共存。

### 🏗️ 架构概览

Sooda 采用**解释器模式**。用户的“流程图”被序列化为 JSON DSL，由后端的 **FlowEngine** 实时解析并执行，避免了传统编译型语言的部署延迟。

```
graph LR
    A[浏览器 Canvas] -->|HTTP| B[Nginx]
    B --> C{网关路由}
    C -->|静态资源| D[Web Server]
    C -->|API请求| E[FlowEngine 核心]
    E --> F[DB 连接池]
    E --> G[第三方服务]
    E --> H[缓存层]
```

### 🚀 快速开始

1. **克隆仓库**

```
git clone https://github.com/yourname/sooda.git
cd sooda
```

2. **安装依赖**

```
# 以 Node.js/React 技术栈为例
cd web && npm install
cd ../engine && npm install
```

3. **启动服务**

```
npm run dev
# 前端画布: http://localhost:3000
# API 服务: http://localhost:8080
```

4. **体验**
打开画布，拖拽一个“模拟数据”节点并发布，立刻在浏览器中访问生成的 URL，感受“秒级生效”的快感。

```
├── /connectors   # 各类数据源连接器

```
